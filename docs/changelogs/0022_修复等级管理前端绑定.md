# 变更日志: 修复等级管理前端绑定
**日期**: 2025-11-27 17:00
**任务**: 解决等级管理页按钮未携带 ID 导致的 500 错误

### 修改 (Modified)
- `views/grade-management.html`: 渲染等级列表时统一使用 `grade.id ?? grade._id`，并通过事件监听绑定“编辑/删除”按钮，兼容新旧字段（`badgeUrl/badge_url`），避免出现 `/api/grades/undefined` 请求。

### 新增 (Created)
- `docs/changelogs/0022_修复等级管理前端绑定.md`: 记录此次等级管理前端修复。

### 删除 (Deleted)
- 无

## 💡 技术说明 (Technical Notes)
- 采用 DOM 事件绑定替代内联 `onclick`，可安全处理包含特殊字符的名称，并确保 N 次渲染后仍能准确引用对应的记录 ID。
